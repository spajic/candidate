% АКТУАЛЬНОСТЬ ТЕМЫ ИССЛЕДОВАНИЯ ----------------------------------------------
	\mkcommonsect{actuality} {Актуальность темы исследования.}{%
	%l: 1. ЕСГ управляется АСДУ
	Единая система газоснабжения (ЕСГ) России управляется 
	распределённой иерархической
	Автоматизированной Системой Диспетчерского Управления (АСДУ).

	%l: 2. В составе АСДУ важная часть - ПВК
	В работе диспетчерских служб (ДС) всех уровней активно применяются 
	многочисленные программно-вычислительные комплексы (ПВК) 
	систем поддержки принятия диспетчерских решений (СППДР), 
	основанные на моделировании режимов систем газоснабжения (СГ).

	%l: 3. Газпром модернизирует АСДУ
	В настоящее время ОАО «Газпром» реализует проект модернизации АСДУ ЕСГ России.
	Для информационного обеспечения всех подсистем АСДУ создаются 
	централизованные источники актуализированных данных и 
	распределённые средства доступа к ним~--- 
	единое информационное пространство (ЕИП).
	Для обеспечения вычислительных потребностей АСДУ создан ряд 
	мощных центров обработки данных (ЦОД).

	%l: 4. Новые требования к подсистеме моделирования, ПВК
	Новые требования предъявляются и к ПВК СППДР:
	\begin{itemize}
		\item
			быстрое решение наиболее вычислительно-сложных задач моделирования СГ
			на основе параллельных и распределённых вычислений и с 
			использованием всех доступных вычислительных ресурсов: от
			персональных компьютеров ДС до ЦОД;
		\item
			работа в многопользовательском, распределённом режиме, соответствующем
			иерархии АСДУ;
		\item
			интеграция с ЕИП;
		\item
			обеспечение согласованности результатов работы различных ПВК,
			их взаимодействие на основе стандартных протоколов информационного обмена.
	\end{itemize}

	%l: 5. Факторы, затрудняющие реализацию новых требований.
	Реализацию перечисленных требований кардинально затрудняет ряд факторов:
	\begin{itemize}
		\item
			использование в каждом ПВК собственного расчётного модуля и взаимодействие
			с ним по закрытому интерфейсу затрудняет согласование результатов
			работы ралзичных ПВК, их совместное использование, интеграцию с ЕИП;
		\item
			ориентация расчётных модулей на работу в рамках одного компьютера, 
			последовательную организацию вычислительных процессов;
		\item 
			монолитность архитектуры расчётных модулей ~--- 
			длительная разработка расчётных модулей 
			на основе процедурного подхода привела~к
			высокой степени взаимопроникновения между их элементами, 
			что крайне затрудняет осуществление модернизации.
	\end{itemize}
	
	%l: 6. Путь решения - создание единой вычислительной среды.
	Один из путей осуществления необходимой модернизации~--- разработка
	унифицированного обеспечения иерархической, распределенной СППДР
	программно-вычислительными ресурсами для решения задач ДС всех уровней~--- 
	единого вычислительного пространства (ЕВП) АСДУ ЕСГ.

	%l: 7. Характеристика такой среды
	Разработку ЕВП целесообразно вести на базе единой вычислительной среды,
	обеспечивающей все средства	для создания вычислительных сервисов,
	отвечающих новым требованиям к вычислительному обеспечению СППДР.

%l: ЦЕЛИ И ЗАДАЧИ ДИССЕРТАЦИОННОЙ РАБОТЫ --------------------------------------
	\mkcommonsect{objective}{Цели и задачи диссертационной работы:}{%
		%l: Цель - разработка единой вычислительной среды, базовой для разработки
		% вычислительных сервисов ПВК СППДР АДСУ ЕСГ России.		
		Целью диссертационной работы является 
		  разработка единой вычислительной среды 
		  решения задач моделирования режимов систем газоснабжения 
		  на основе применения технологий параллельных и распределённых вычислений.

		Для достижения поставленной цели были решены следующие задачи:
		\begin{enumerate}
			\item % 1. Анализ АСДУ ЕСГ, ЕИП, ПВК моделиров-я СГ
			Анализ текущего состояния и направлений развития АСДУ ЕСГ России, 
			единого информационного пространства системы диспетчерского управления ЕСГ, 
			архитектур построения ПВК моделирования режимов систем газоснабжения (СГ).
			\item % 2. Проектирование среды, алгоритмов
			Проектирование компонентной среды организации параллельных и распределённых 
			вычислительных процессов решения задач моделирования режимов СГ и алгоритмов 
			создания вычислительных сервисов в этой среде.
			\item % 3. Арх-ра выч сервисов моделирования СГ и адаптации (идентиф-ии)
			Разработка архитектурных решений для вычислительных сервисов решения задач 
			моделирования режимов СГ, адаптации модели режимов СГ к фактическим режимам.
			\item % 4. Программная реализация ядра среды, ВС моделир-я СГ и идент-ии
			Программная реализация ядра предложенной среды и вычислительных сервисов 
			моделирования режимов систем газоснабжения.
			\item % 5. Проверка корректности работы и оценка произв-ти разр-го ПО
			      % в различных сценариях, режимах: однопольз, сетевом, распр-м
			Проверка корректности и оценка производительности разработанного ПО 
			в однопользовательском, сетевом, распределённом режимах.
		\end{enumerate}
} % Цели и задачи диссертационной работы (конец) ------------------------------
